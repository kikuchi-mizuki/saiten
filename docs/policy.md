# 教授コメント自動化ボット 利用規約

**最終更新**: 2025-11-11
**バージョン**: 1.0

---

## 1. サービス概要

本サービス（以下「本システム」）は、経営戦略論の教授がレポートコメント作成を効率化するためのAIアシスタントツールです。

### 主な機能
- レポート本文に対するコメント自動生成
- Rubric自動採点（5項目、1-5点）
- レポート要約生成
- 過去のコメント履歴管理
- コメント品質評価（手直し時間測定、満足度アンケート）

---

## 2. 利用対象者

本システムは、経営戦略論の担当教授のみが利用できます。

---

## 3. 個人情報の取り扱い

### 3.1 収集する情報

本システムは、以下の情報を収集・保存します：

| 情報 | 目的 | 保存期間 |
|------|------|---------|
| Googleアカウント情報（メールアドレス） | 認証・ユーザー識別 | 無期限（削除可能） |
| レポート本文 | コメント生成・履歴保存 | 無期限（削除可能） |
| 生成されたコメント | 履歴保存・品質評価 | 無期限（削除可能） |
| Rubric採点結果 | 履歴保存・統計分析 | 無期限（削除可能） |
| 手直し時間 | 品質評価・改善 | 無期限（削除可能） |
| 満足度評価 | 品質評価・改善 | 無期限（削除可能） |

### 3.2 データの保存場所

- **データベース**: Supabase (PostgreSQL、クラウドホスティング)
- **リージョン**: Asia Pacific (Tokyo)
- **暗号化**: レポート本文はAES-256-GCMで暗号化して保存

### 3.3 データの保存期間

- **原則**: 無期限保存（ユーザーが削除しない限り）
- **削除**: ユーザーはいつでも履歴画面から個別にデータを削除可能
- **削除方法**: 履歴詳細画面の「削除」ボタンをクリック → 即座に物理削除

### 3.4 個人情報保護対策

本システムは、以下のセキュリティ対策を実施しています：

- **認証**: Google OAuth 2.0による安全な認証
- **データ暗号化**: レポート本文をAES-256-GCMで暗号化
- **アクセス制御**: Row Level Security (RLS)によるユーザー毎のデータ分離
- **通信暗号化**: HTTPS/TLSによる通信の暗号化
- **PII検出・マスキング**: レポート内の個人情報（氏名、学籍番号、メールアドレス、電話番号）を自動検出・マスキング

## 4. 第三者へのデータ提供

本システムは、以下の第三者サービスにデータを送信します：

### 4.1 OpenAI API（必須）

**提供先**: OpenAI, L.L.C.
**提供データ**: レポート本文（PII検出後、個人情報をマスキングしたもの）
**提供目的**: コメント生成・Rubric採点・要約生成
**プライバシーポリシー**: https://openai.com/policies/privacy-policy

**OpenAIのデータ取り扱い**:
- APIリクエストのデータは30日間保持される
- 学習データには使用されない（API利用規約による）
- 詳細はOpenAIのプライバシーポリシーを参照

### 4.2 Supabase（必須）

**提供先**: Supabase, Inc.
**提供データ**: すべてのユーザーデータ
**提供目的**: データベース・認証サービスの提供
**プライバシーポリシー**: https://supabase.com/privacy

### 4.3 Vercel（必須）

**提供先**: Vercel Inc.
**提供データ**: アクセスログ
**提供目的**: フロントエンドホスティング
**プライバシーポリシー**: https://vercel.com/legal/privacy-policy

## 5. データの利用目的

収集したデータは、以下の目的でのみ利用します：

1. **コメント生成**: レポート本文からコメントを生成
2. **履歴管理**: 過去のコメントを保存・閲覧
3. **品質評価**: 手直し時間・満足度の統計分析
4. **システム改善**: サービス品質向上のための分析
5. **研究目的**: 教授による教育研究（匿名化後）

**第三者への提供**: 上記4項に記載の場合を除き、ユーザーの同意なく第三者に提供することはありません。

## 6. ユーザーの権利

### 6.1 データの閲覧

ユーザーは、履歴画面からいつでも自分のデータを閲覧できます。

### 6.2 データの削除

ユーザーは、以下の方法でデータを削除できます：

- **個別削除**: 履歴詳細画面の「削除」ボタン → 即座に物理削除
- **一括削除**: （Phase 2-Aで実装予定）

### 6.3 データのエクスポート

ユーザーは、履歴画面の「エクスポート」ボタンから、すべてのデータをJSON/CSV形式でダウンロードできます。
