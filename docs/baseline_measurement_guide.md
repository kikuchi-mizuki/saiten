# ベースライン測定ガイド

**目的**: AI不使用時の作業時間を測定し、AI使用時との効果を定量的に比較するためのベースラインデータを取得する
**実施時期**: UAT実施前（Week 12 Day 2）
**所要時間**: 約2時間
**測定対象**: 5件のレポートに対するコメント作成作業

## 🎯 測定目的

### 主要目的
AI活用による以下の効果を定量的に測定するため、AI不使用時のベースラインデータを取得する：

1. **作業時間削減効果**: AI使用時と比較して、どれだけ作業時間が削減されるか
2. **品質維持**: AI使用時でも、手動作成と同等以上の品質が維持されているか
3. **効率向上**: 総合的な作業効率の改善度合い

### 測定対象の作業
教授が通常行っている以下の作業を測定：

1. **レポートを読む**: レポート本文を読み、内容を理解する
2. **Rubric採点**: 5項目（理解度、論理性、独自性、実践性、表現力）を1-5点で採点
3. **コメント作成**: 過去のコメント例を参照しながら、300-400文字程度のコメントを作成
4. **推敲・修正**: コメントを読み直し、必要に応じて修正

## 📐 測定方法

### 測定環境
- **場所**: 教授の通常作業環境（研究室またはオフィス）
- **ツール**: ストップウォッチ（スマートフォンのタイマーアプリ等）
- **参照資料**: 過去のコメント例（`data/sample_comments.json`を印刷したもの、または画面表示）

### 測定条件
- **レポート数**: 5件（reflection 3件、final 2件）
- **文字数範囲**: 300〜600文字程度
- **中断**: 測定中は他の作業を行わず、集中してコメント作成を行う
- **参照**: 過去のコメント例を自由に参照可能

### 注意事項
- できるだけ通常の作業ペースで行う（速すぎず、遅すぎず）
- 途中で中断した場合は、その時間を記録から除外する
- コメントの品質は通常と同等レベルを維持する

## 📊 測定項目

### 1. 時間データ

| 項目 | 内容 | 測定単位 |
|------|------|---------|
| **読む時間** | レポート本文を読み、内容を理解する時間 | 秒 |
| **Rubric採点時間** | 5項目を採点する時間 | 秒 |
| **コメント作成時間** | コメントを執筆する時間 | 秒 |
| **推敲・修正時間** | コメントを見直し、修正する時間 | 秒 |
| **総作業時間** | 上記の合計時間 | 秒 |

### 2. 品質データ

| 項目 | 内容 |
|------|------|
| **コメント文字数** | 作成したコメントの文字数 |
| **参照例の使用数** | 参照した過去コメント例の数 |
| **修正回数** | コメントを修正した回数 |

### 3. 主観評価データ

| 項目 | 内容 | 評価尺度 |
|------|------|---------|
| **作業負荷** | 作業の負担度 | 1-5（1: 軽い、5: 重い） |
| **難易度** | コメント作成の難易度 | 1-5（1: 簡単、5: 難しい） |
| **満足度** | 作成したコメントへの満足度 | 1-5（1: 低い、5: 高い） |

## 📝 測定手順

### 事前準備（10分）

1. **レポートサンプルの準備**
   - 5件のレポートを印刷またはPDF化
   - レポートNo.を付与（B1〜B5）
   - 内訳: reflection 3件（B1, B2, B3）、final 2件（B4, B5）

2. **過去コメント例の準備**
   - `data/sample_comments.json`から10-15件を印刷または画面表示
   - reflection用とfinal用を分けて整理

3. **記録用紙の準備**
   - 時間記録シート（後述のフォーマット参照）
   - ストップウォッチまたはタイマーアプリ

### 測定実施（各レポート15-25分 × 5件 = 75-125分）

**各レポートに対して以下の手順を実施**:

#### ステップ1: 読む時間の測定
1. タイマーをスタート
2. レポート本文を読み、内容を理解する
3. 理解が完了したらタイマーを停止
4. 経過時間を記録シートに記入

#### ステップ2: Rubric採点時間の測定
1. タイマーをスタート
2. 5項目（理解度、論理性、独自性、実践性、表現力）を1-5点で採点
3. 各項目の採点理由を簡潔にメモ
4. 採点完了したらタイマーを停止
5. 経過時間を記録シートに記入

#### ステップ3: コメント作成時間の測定
1. タイマーをスタート
2. 過去のコメント例を参照しながら、コメントを執筆
3. 目標文字数: 300-400文字程度
4. 初稿が完成したらタイマーを一時停止
5. 経過時間を記録シートに記入

#### ステップ4: 推敲・修正時間の測定
1. タイマーをスタート（再開）
2. コメントを読み直し、必要に応じて修正
3. 最終的なコメントが完成したらタイマーを停止
4. 経過時間を記録シートに記入

#### ステップ5: 品質データと主観評価の記録
1. コメント文字数をカウント
2. 参照した過去コメント例の数を記録
3. 修正回数を記録
4. 作業負荷、難易度、満足度を評価（1-5点）

#### ステップ6: 休憩（5分）
- 次のレポートに進む前に、5分程度の休憩を取る
- ステップ1〜5を残り4件のレポートで繰り返す

### 測定完了後（10分）

1. **データの集計**
   - 5件の総作業時間を合計
   - 平均作業時間を計算（総作業時間 ÷ 5）
   - 各ステップの平均時間を計算

2. **記録シートの保存**
   - 記録シートをスキャンまたは写真撮影
   - `docs/baseline_measurement_data.csv`として保存

## 📈 分析方法

### 1. 平均作業時間の算出

**計算式**:
```
平均総作業時間 = Σ(各レポートの総作業時間) ÷ 5

平均ステップ別時間:
- 平均読む時間 = Σ(各レポートの読む時間) ÷ 5
- 平均Rubric採点時間 = Σ(各レポートのRubric採点時間) ÷ 5
- 平均コメント作成時間 = Σ(各レポートのコメント作成時間) ÷ 5
- 平均推敲・修正時間 = Σ(各レポートの推敲・修正時間) ÷ 5
```

**期待値**:
- 平均総作業時間: 約300-500秒（5-8分）/ 1件

### 2. レポート種別ごとの比較

**reflection vs final の比較**:
```
reflection平均時間 = (B1 + B2 + B3) ÷ 3
final平均時間 = (B4 + B5) ÷ 2
```

**期待される傾向**:
- final の方が reflection より時間がかかる（レポートが長く、総括的な内容のため）

### 3. AI使用時との比較（UAT実施後）

**削減時間の計算**:
```
削減時間 = ベースライン総作業時間 - AI使用時総作業時間
削減率 = (削減時間 ÷ ベースライン総作業時間) × 100%
```

**目標削減率**: 30%以上

**例**:
- ベースライン総作業時間: 400秒（6分40秒）
- AI使用時総作業時間: 280秒（4分40秒）
  - 生成時間: 20秒
  - 手直し時間: 260秒（4分20秒）
- 削減時間: 120秒（2分）
- 削減率: 30%

### 4. 品質維持の確認

**コメント文字数の比較**:
```
ベースライン平均文字数 vs AI生成平均文字数
```

**期待される結果**:
- AI生成コメントも同程度の文字数（300-400文字程度）
- 文体や内容の質が同等以上

### 5. 主観評価の比較

**作業負荷の比較**:
```
ベースライン平均作業負荷 vs AI使用時平均作業負荷
```

**期待される結果**:
- AI使用時の方が作業負荷が低い（2-3点程度）

## 📝 測定結果の活用

### UAT報告書への記載
ベースライン測定結果は、UAT報告書の以下のセクションに記載：

1. **ベースライン測定結果**:
   - 平均総作業時間: XX分XX秒
   - ステップ別平均時間
   - レポート種別ごとの平均時間

2. **AI使用時との比較**:
   - 削減時間: XX分XX秒
   - 削減率: XX%
   - 目標達成状況（30%以上削減）

3. **品質比較**:
   - コメント文字数の比較
   - 主観評価の比較

### Phase 2以降への示唆
- ベースライン測定で明らかになった時間のかかる作業を、Phase 2でさらに効率化
- 例: コメント作成時間が最も長い場合、Phase 2-AでRAG精度を向上させる
